# 2. Visão técnica do projeto

O My-co-Lab começou como um sistema pessoal de monitorização ambiental para cultivo de cogumelos, com o objetivo de evoluir para uma plataforma modular de cultivo inteligente.

A primeira versão funcional tem como objetivo medir temperatura e humidade, mostrar informação localmente, guardar dados e permitir exportação offline.

O sistema foi pensado para funcionar mesmo sem internet.

Objetivos técnicos principais:

* medir temperatura
* medir humidade
* mostrar dados num LCD
* guardar dados em CSV
* funcionar em modo headless
* funcionar offline
* exportar dados automaticamente para uma PEN USB
* manter cópia do projeto em GitHub
* preparar base futura para ventoinha, humidificador, LEDs, câmara e timelapse

# 14. Problemas encontrados no backup USB

## 14.1 USB inserida sem reação

Investigação:

```bash
lsblk
```

Resultado observado:

```text
sda1
/media/pi/E7E6-258B
```

A PEN estava correta.

Depois foi procurado o script:

```bash
find ~/My-co-Lab -type f | grep -i backup
```

Descoberta:

O script tinha sido movido para:

```text
/home/pi/My-co-Lab/scripts/production/usb_backup.sh
```

Mas o serviço ainda apontava para:

```text
/home/pi/My-co-Lab/scripts/usb_backup.sh
```

Problema:

Path hardcoded desatualizado.

Solução:

Editar serviço:

```bash
sudo nano /etc/systemd/system/mycolab-usb-backup@.service
```

Substituir:

```ini
ExecStart=/home/pi/My-co-Lab/scripts/usb_backup.sh %I
```

por:

```ini
ExecStart=/home/pi/My-co-Lab/scripts/production/usb_backup.sh %I
```

Depois correr:

```bash
sudo systemctl daemon-reload
```

Lição:

Sempre que scripts são movidos, atualizar serviços systemd associados.

# 15. Problemas gerais encontrados

## 15.1 mpremote bloqueado

Erro:

```text
/dev/ttyACM0 in use
```

Causa:

O logger estava a usar a porta serial.

Investigação:

```bash
sudo lsof /dev/ttyACM0
```

Solução:

Parar o logger antes de usar mpremote.

Exemplo:

```bash
sudo systemctl stop mycolab-logger.service
python3 -m mpremote connect /dev/ttyACM0
```

Depois:

```bash
sudo systemctl start mycolab-logger.service
```

Lição:

A porta serial só pode ser usada por um processo de cada vez.

---

## 15.2 LCD preso em BACKUP DONE

Problema:

Após backup, LCD ficava preso na mensagem:

```text
BACKUP DONE
```

Solução:

Reiniciar Pico após backup.

Comando:

```bash
python3 -m mpremote reset
```

Lição:

Após operações externas no LCD/Pico, o sistema precisa de voltar ao estado normal.

---

## 15.3 CSV em pasta errada

Problema:

Ficheiros apareceram em:

```text
timelapse/environment/
```

em vez de:

```text
data/environment/
```

Solução:

Corrigir comando rsync e estrutura de destino.

Lição:

A estrutura de pastas precisa ser consistente desde cedo.

---

## 15.4 Dados apagados localmente

Objetivo:

Depois do backup, limpar dados locais para evitar encher cartão SD.

Verificação:

```bash
ls ~/My-co-Lab/data/environment
```

Lição:

Limpeza automática é útil, mas só deve acontecer depois de confirmar que a cópia foi bem sucedida.
